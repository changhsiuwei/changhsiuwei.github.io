## 先看門把，不要先看模型

辦公室自動化以前很好理解。表單送出，系統寄信；資料進來，試算表更新；每天早上九點，報表寄到主管信箱。這些流程笨，但笨得可靠。它們照規則走，不會忽然自己決定去翻你的資料夾，也不會因為覺得任務還沒完成，就幫你多寄三封信。Agentic AI 麻煩的地方，正在於它看起來比較聰明。

**真正的問題是交界處**

它可以接受一個目標，拆成步驟，呼叫工具，讀檔案，查網頁，修改表格，檢查結果，再進下一步。這聽起來像知識工作者等了很久的自動化。可是只要一個系統能自己往前走，問題就不再是它會不會做，而是它知不知道哪裡不能自己做。很多人談 agent 時，喜歡講效率。我比較想先講門把。哪些門可以讓它自己開？

哪些門只能敲？哪些門它連碰都不該碰？讀公開網頁和讀私人信箱不是同一件事。整理草稿和寄出正式信件不是同一件事。幫你找資料和替你做決策也不是同一件事。若邊界沒有先畫好，agent 的能力越強，風險越大。一個可用的 agent 工作流至少有幾個零件。語言模型負責理解目標與產生下一步。

工具介面讓它接觸檔案、瀏覽器、試算表、資料庫或 API。記憶系統記住已完成的動作、偏好與錯誤。監督機制決定何時要停下來問人。少了最後一個，整套系統就像一個很勤快但沒有分寸的實習生。

## 會自己往前走的系統

「沒有分寸」不是小問題。假設你叫 agent 幫你整理會議資料。它找到舊檔案，讀了簡報，生成摘要，這很好。接著它看到某個資料缺漏，自己上網找了一份來源不明的表格補進去。再接著，它把摘要寄給所有與會者。流程看起來完成了，但你可能已經把錯資料正式送出。

**不要讓系統假裝確定**

這種錯誤不是模型單一句子幻覺，而是行動幻覺。它不只說錯，還做錯。因此，Agentic AI 的設計重點不是讓它一路自動到底，而是安排停頓。什麼情況要人工核准？牽涉外部寄送要不要停？修改原始檔要不要先另存副本？抓到新資料要不要標示來源與信心？遇到衝突資料要不要停止，而不是選一個看起來合理的？

這些問題很無聊，但真正決定系統能不能用。教育現場也該教這件事。學生未來面對的 AI，不會只是聊天框，而是一堆能操作工具的系統。與其只教他們寫提示詞，不如讓他們設計一個小型工作流：目標是什麼？輸入資料在哪裡？哪些步驟可以自動化？哪些地方必須由人確認？錯了怎麼回復？

這比單純比較哪個模型回答比較好，更接近未來的工作。Agentic AI 也會逼我們重新看「責任」。過去流程錯了，可以查程式規則。人做錯了，可以問那個人。agent 介於中間：它照著模型判斷走，又呼叫了工具，又可能根據記憶改變下一步。若沒有日誌，事後很難知道錯在哪裡。

可追蹤不是工程細節，而是治理底線。每一步做了什麼、用了哪些資料、誰核准、何時修改，都要留下來。

## 停頓是一種設計

我不反對 agent。相反，我覺得它會吃掉大量辦公室裡最磨人的工作：整理附件、比對版本、生成初稿、更新表格、檢查缺漏。這些事值得被機器接手。只是越值得接手，越要先設計好剎車。好用的 agent 不該像一台停不下來的車。它要像一個懂規矩的助手：能自己處理小事，遇到會傷人的事就停下來看你。

**權限要比能力更清楚**

如果把 Agentic AI 當成「完全無人化」，那會很快出事。如果把它當成「有人負責的自動化」，它才可能進辦公室。未來的問題不是機器能不能替我們做更多，而是我們能不能清楚說出：到哪一步為止，它必須把手放開。權限設計可以很土，但一定要有。第一層，只能讀不能寫。

第二層，可以產生草稿但不能覆蓋原檔。第三層，可以修改副本但需要人工合併。第四層，牽涉外部寄送、付款、刪除、公開發布，一律停下來等人。這些規則看起來像把聰明系統綁住，其實是在保護它能進入真實工作。沒有邊界的自動化，很快會被組織排斥。記憶也是雙面刃。agent 記得你的偏好，工作會變快；

它記得太多，風險也變大。哪些記憶可以保存？保存多久？能不能刪？是否包含學生資料、客戶資料、研究草稿？這些問題不能等出事才問。很多人談 agent 時，只談它能不能完成任務，不談它帶著哪些記憶完成任務。可是知識工作最敏感的部分，常常就在那些脈絡裡。

## 日誌不是裝飾

我最想看到的 agent，不是那種自稱能替你處理一切的系統，而是會誠實說「這一步我不能自己做」的系統。它會把資料整理好，把選項列出來，把風險標出來，然後停在門口等你。這種停頓看似降低效率，實際上是信任的來源。人願意交出更多工作，不是因為機器永遠往前衝，而是因為它知道哪些地方必須回頭看人。

**把黑箱拆成可追問的步驟**

組織導入 agent 時，最好先挑那些「錯了也能回復」的流程。整理會議紀錄、產生草稿、比對檔案版本，這些適合試。直接更新正式資料庫、寄送外部信件、刪除檔案、處理個資，這些要晚一點。不是因為技術做不到，而是因為組織還沒有足夠的肌肉處理錯誤。自動化最怕一開始就碰不可逆的事。

還要安排人定期閱讀日誌。很多系統會留下紀錄，但沒有人看。沒有人看的紀錄，只是安慰。每週抽幾個任務，檢查 agent 做了哪些步驟、在哪裡停下、用了哪些資料、是否有不必要的工具呼叫。這些檢查會讓團隊知道系統正在形成什麼習慣。人類管理者不能只看結果，也要看路徑。

因為 agent 真正危險的地方，常常藏在那條看似順利的路徑裡。如果要給 agent 一句工作守則，我會寫：能準備，不要擅自決定；能整理，不要偷偷發布；能建議，不要直接承諾。這三句話很保守，但辦公室需要保守。知識工作裡最麻煩的不是少做一步，而是多做了一步不可挽回的事。

## 讓人睡得著的自動化

真正成熟的自動化，應該讓人睡得著。它不是半夜突然替你寄出文件，也不是自己改掉原始資料，而是在你隔天打開電腦時，把草稿、差異、風險和待核准事項排好。你仍然要工作，但工作變得比較像判斷，而不是搬運。這才是 agent 進辦公室比較值得期待的樣子。
